# Praktika — тренажёр диагностического профиля

Веб-приложение для подготовки к практике по диагностическому профилю (2 курс,
лечебный факультет). Позволяет учить вопросы в формате карточек, отмечать, что уже
выучено, и отслеживать прогресс на дашборде.

- **64 темы**, ~315 вопросов с эталонными ответами (из методички).
- Формат карточек: вопрос → показать эталонный ответ → отметить «Знаю» / «Повторить».
- Дашборд с общим прогрессом, статистикой и списком тем.
- Прогресс сохраняется локально (в браузере) **или** синхронизируется через Supabase,
  если он подключён.

Стек: **Vite + React + TypeScript**, **Zustand** (состояние), **Supabase** (бэкенд,
опционально), **lucide-react** (иконки). Тёмная тема, без внешних зависимостей во
время выполнения.

## Быстрый старт

```bash
npm install
npm run dev
```

Откройте адрес, который выведет Vite (обычно http://localhost:5173).

Приложение сразу работает в **локальном режиме** — прогресс хранится в `localStorage`
этого браузера. Аккаунт и синхронизация не обязательны.

Сборка продакшн-версии:

```bash
npm run build      # проверка типов + сборка в dist/
npm run preview    # локальный предпросмотр собранной версии
```

## Подключение Supabase (опционально)

Нужно, только если хотите аккаунты и синхронизацию прогресса между устройствами.

1. Создайте бесплатный проект на [supabase.com](https://supabase.com).
2. В **SQL Editor** выполните скрипт [`supabase/schema.sql`](supabase/schema.sql)
   (создаёт таблицу `progress` и политики RLS).
3. Скопируйте `.env.example` в `.env` и заполните значения из
   **Project Settings → API**:

   ```env
   VITE_SUPABASE_URL=https://xxxx.supabase.co
   VITE_SUPABASE_ANON_KEY=eyJhbGc...
   ```

4. Перезапустите `npm run dev`. В боковой панели появится вход/регистрация.

Если переменные не заданы, приложение автоматически остаётся в локальном режиме.

### Как устроена синхронизация

- Каждая карточка хранится строкой `(user_id, question_id, status, reviews, updated_at)`.
- При входе локальный и удалённый прогресс объединяются (побеждает более свежая
  запись по `updated_at`), затем локальные изменения выгружаются на сервер.
- Row Level Security гарантирует, что пользователь видит и меняет только свои строки.

## Структура проекта

```
src/
  data/            # вопросы по темам (part1..part8) + сборка в topics.ts
  lib/             # типы, supabase-клиент, статистика, форматирование
  store/           # zustand-хранилище прогресса (localStorage + Supabase)
  auth/            # контекст авторизации
  components/      # Sidebar, Topbar, карточки, модалки, кольцо прогресса
  pages/           # Dashboard, Topics, Study, ProgressPage
supabase/
  schema.sql       # схема БД и политики RLS
```

## Как редактировать вопросы

Вопросы лежат в `src/data/part1.ts … part8.ts` в компактном формате:

```ts
{
  id: 1,
  title: "Оценка состояния пациента",
  questions: [
    // строка в `a` => ответ-определение; массив => ответ-перечисление
    { q: "Перечислите варианты состояния пациента.", a: ["Удовлетворительное", "…"] },
    { q: "Дайте определение ступора.", a: "Состояние оглушения, …" },
  ],
}
```

`buildTopics` в `src/data/build.ts` автоматически проставляет `id` вопросов и
определяет тип ответа. Ничего пересобирать вручную не нужно — просто отредактируйте
нужный файл.

## Статусы вопроса

- **Не начато** (`new`) — карточку ещё не открывали.
- **На повторении** (`learning`) — отметили «Повторить».
- **Выучено** (`learned`) — отметили «Знаю». Идёт в общий прогресс.

> Медицинский контент взят из учебной методички и предназначен только для подготовки
> к практике, а не для клинических решений.
